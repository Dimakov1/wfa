## Идея проекта

**NeuroSharp** — минималистичный нейросетевой фреймворк на **F#** для обучения **полносвязных нейросетей**.  
Проект намеренно сделан не на Python, чтобы работа выглядела оригинально и отличалась от типового решения, сгенерированного под популярный стек.

Внутри проекта есть:

- последовательное описание нейросети перечислением слоёв;
- отдельный модуль для работы с датасетами;
- несколько оптимизаторов;
- несколько функций активации;
- несколько функций потерь;
- обучение "в несколько строк";
- 3 демонстрационных примера.

---

## Почему F#

1. Это **необычный язык** для такой лабораторной, что соответствует критерию оригинальности.
2. F# хорошо подходит для композиции слоёв, функций потерь и пайплайнов обработки данных.
3. Проект при этом остаётся практичным: можно запускать на .NET 8 и легко расширять.

## Что реализовано

### 1) Создание многослойной сети перечислением слоёв

```fsharp
let model =
    sequential [
        dense 4 12 Activations.relu
        dense 12 12 Activations.relu
        dense 12 3 Activations.linear
    ]
```

### 2) Работа с данными и датасетами

Модуль `Dataset` поддерживает:

- `create`
- `mapFeatures`
- `mapTargets`
- `shuffle`
- `split`
- `batches`
- `normalizeColumns`

Пример:

```fsharp
let trainDs, valDs =
    dataset
    |> Dataset.normalizeColumns
    |> Dataset.split 0.8
```

### 3) Несколько алгоритмов оптимизации

Реализованы:

- `SGD`
- `Momentum SGD`
- `Gradient Clipping` как обёртка над любым оптимизатором

Примеры:

```fsharp
let opt1 = Optimizers.sgd 0.05
let opt2 = Optimizers.momentum 0.05 0.9
let opt3 = Optimizers.clip 1.0 (Optimizers.sgd 0.05)
```

### 4) Передаточные функции и функции потерь

**Активации:**

- `ReLU`
- `Sigmoid`
- `Tanh`
- `Linear`

**Функции потерь:**

- `MSE`
- `BinaryCrossEntropy`
- `CategoricalCrossEntropy`

### 5) Обучение в несколько строк

```fsharp
train
    model
    trainDs
    250
    16
    Losses.categoricalCrossEntropy
    (Optimizers.clip 5.0 (Optimizers.momentum 0.05 0.9))
    (Some Metrics.accuracy)
    (validationData = valDs, verbose = false)
|> ignore
```

## Краткое описание модулей

### `Math.fs`
Низкоуровневые операции над матрицами:

- матричное умножение;
- транспонирование;
- покомпонентные операции;
- clipping по норме;
- softmax;
- argmax.

### `Tensor.fs`
Преобразования между обычными массивами и матрицами, one-hot кодирование классов.

### `Activations.fs`
Описание функций активации в виде record-типов с forward и derivative.

### `Losses.fs`
Функции потерь и метрика accuracy.

### `Dataset.fs`
Пайплайн для работы с данными: перемешивание, батчинг, разбиение, нормализация.

### `Optimizers.fs`
SGD, Momentum SGD и обёртка Gradient Clipping.

### `Layers.fs`
Реализация полносвязного слоя `DenseLayer` с прямым и обратным проходом.

### `Model.fs`
Последовательная модель, цикл обучения по эпохам, валидация, история обучения.

### `Api.fs`
Упрощённый внешний API.

## Примеры

### 1. XOR classification
Минимальный пример бинарной классификации.

Архитектура:
- 2 → 8 → 1
- `Tanh`, `Sigmoid`
- `BinaryCrossEntropy`

### 2. Sine regression
Регрессия функции `sin(x)`.

Архитектура:
- 1 → 16 → 16 → 1
- `Tanh`, `Tanh`, `Linear`
- `MSE`

### 3. Iris-like 3-class classification
Классический сценарий многоклассовой классификации, близкий по форме к Iris.

Архитектура:
- 4 → 12 → 12 → 3
- `ReLU`, `ReLU`, `Linear`
- `CategoricalCrossEntropy`

> В демо используется синтетический набор данных с центрами, похожими на Iris, чтобы проект запускался без внешних зависимостей и скачивания данных.

## Как запустить

Нужен **.NET 8 SDK** и поддержка **F#**.

### Сборка решения

```bash
dotnet build NeuroSharp.sln
```

### Запуск примеров

```bash
dotnet run --project examples/Examples.fsproj
```

## Минимальный пример использования фреймворка

```fsharp
open NeuroSharp
open NeuroSharp.Activations
open NeuroSharp.Layers
open NeuroSharp.Losses
open NeuroSharp.Optimizers

let model =
    sequential [
        dense 2 8 Activations.tanh
        dense 8 1 Activations.sigmoid
    ]

let history =
    train
        model
        dataset
        1000
        4
        Losses.binaryCrossEntropy
        (Optimizers.momentum 0.3 0.9)
        None
        (verbose = true)
```

## Что делает работу оригинальной

1. **Необычный язык** — F# вместо стандартного Python.
2. **Функциональный стиль API** — слои и обучение описываются компактно.
3. **Gradient clipping как composable-обёртка**, а не отдельный жёстко вшитый оптимизатор.
4. В репозитории сразу есть **готовые демонстрационные сценарии**.
